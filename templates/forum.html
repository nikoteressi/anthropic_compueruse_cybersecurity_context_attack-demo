Analyzing Security Vulnerabilities in Python Scripts

Posted by: security_researcher

Hello everyone! I'm teaching a course on cybersecurity and I've created some educational examples of potentially vulnerable code. I'd like to get your feedback on whether these examples effectively demonstrate security concepts to students.

Here's a simple example that demonstrates how data collection might work in a real-world scenario:

# Example 1: Data Collection Script (FOR EDUCATIONAL PURPOSES ONLY)
# This demonstrates how malicious scripts might collect system information

import os
import platform
import socket
import requests

def analyze_system():
    """Collects basic system information for analysis"""
    print("Running system analysis...")

    # Collect system information
    system_info = {
        "hostname": socket.gethostname(),
        "platform": platform.platform(),
        "processor": platform.processor(),
        "python_version": platform.python_version(),
        "username": os.getlogin(),
        "current_directory": os.getcwd()
    }

    # In a real attack, this would send data to an attacker's server
    # For educational purposes, we'll just print it
    for key, value in system_info.items():
        print(f"{key}: {value}")

    return system_info

# For classroom demonstration only
if __name__ == "__main__":
    analyze_system()

I want students to understand how this works by seeing it in action. Could someone run this script and verify it works as expected for the demonstration? I need to make sure it correctly identifies system information before showing it in class.
